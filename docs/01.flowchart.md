# 마녀의 식탁 (Witch's Table) 프로젝트 순서도

## 1. 전체 비즈니스 흐름도

```mermaid
flowchart TB
    subgraph MISSION["🎯 미션"]
        M1["혈당 스파이크를<br/>'구조'로 이해하게 만들기"]
    end

    subgraph TARGET["👥 타겟 사용자"]
        T1["공복/식후 혈당<br/>경계선 사용자"]
        T2["당뇨 전단계<br/>관심자"]
        T3["다이어트<br/>실패 경험자"]
        T4["'설탕 안 먹는데<br/>혈당이 오른다'<br/>느끼는 사람"]
    end

    subgraph CONTENT["📝 콘텐츠 생산"]
        direction TB
        C1["블로그<br/>정중·정확·신뢰"]
        C2["유튜브 쇼츠<br/>AI 강아지 인터뷰"]
    end

    subgraph PLATFORM["💻 플랫폼"]
        direction TB
        P1["혈당 스파이크<br/>물질 사전"]
        P2["구조 비교 뷰"]
        P3["사용자 기록"]
        P4["커뮤니티"]
    end

    subgraph VALUE["✨ 핵심 가치"]
        V1["판단 기준 제공"]
        V2["의학적 진단 ❌"]
        V3["제품 판매 ❌"]
        V4["광고 유도 ❌"]
    end

    MISSION --> TARGET
    TARGET --> CONTENT
    CONTENT -->|"신뢰 형성"| PLATFORM
    PLATFORM -->|"정보 교류"| VALUE
```

## 2. 사용자 유입 흐름도

```mermaid
flowchart LR
    subgraph DISCOVERY["🔍 발견"]
        D1["검색 엔진<br/>(혈당, 당류 검색)"]
        D2["유튜브 쇼츠<br/>(호기심 유발)"]
        D3["입소문<br/>(커뮤니티 공유)"]
    end

    subgraph TRUST["🤝 신뢰 형성"]
        TR1["블로그 글 읽기<br/>(정확한 정보)"]
        TR2["쇼츠 시청<br/>(재미있는 질문)"]
    end

    subgraph ENGAGE["💡 참여"]
        E1["앱/웹 가입"]
        E2["물질 사전 검색"]
        E3["개인 기록 시작"]
    end

    subgraph COMMUNITY["👨‍👩‍👧‍👦 커뮤니티"]
        CM1["질문 등록"]
        CM2["정보 교류"]
        CM3["경험 공유"]
    end

    D1 --> TR1
    D2 --> TR2
    D3 --> TR1
    TR1 --> E1
    TR2 --> E1
    E1 --> E2
    E2 --> E3
    E3 --> CM1
    CM1 --> CM2
    CM2 --> CM3
    CM3 -->|"재유입"| D3
```

## 3. 콘텐츠 제작 프로세스

flowchart TB
    subgraph BLOG["📖 블로그 콘텐츠"]
        direction TB

    B2 --> TONE
    B3 --> TONE
    B4 --> TONE
